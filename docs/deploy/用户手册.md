## 3. 部署火焰模型

PoC 推荐使用内置 **火焰模型**（YOLOv8，类别：火、烟）。

### 3.1 WEB 操作

1. 进入 **模型管理**
2. 找到 **火焰模型**
3. 点击 **部署**，选择 GPU 节点
4. 等待状态变为 **运行中**

### 3.2 验证推理服务

```bash
curl -s http://<IP>:5000/actuator/health
# 若已知推理地址（部署后 WEB 或 API 返回）：
curl -s http://<推理IP>:9999/health
```

### 3.3 模型状态未更新（已知现象）

部署成功后 WEB 仍显示「未部署」时，可通过 DEVICE API 修正（需登录 token）：

```bash
# 示例：模型 ID=7，status=1 表示已部署
curl -X PUT "http://<IP>:48080/admin-api/model/7/update" \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"status":1}'
```

## 4. 接入摄像头

### 4.1 RTSP（PoC 推荐）

1. **设备管理 → 添加设备**
2. 类型选摄像头 / RTSP
3. 填写 RTSP 地址，例如：  
   `rtsp://用户名:密码@192.168.1.100:554/stream1`
4. 保存并 **预览**，确认画面正常

### 4.2 GB28181

1. 在 DEVICE 模块配置 SIP 参数
2. 摄像头/NVR 注册到平台国标 ID
3. 在设备列表中确认在线

---

## 5. 创建实时烟火检测任务

1. **算法任务 → 新建 → 实时算法任务**
2. 选择摄像头
3. 绑定 **火焰模型**
4. 参数建议（PoC 起点，可按现场调优）：

| 参数 | 建议值 | 说明 |
|------|--------|------|
| 置信度 | **0.35** 起 | 误报多时可提高到 0.5 |
| 抽帧间隔 | **3** | GPU 紧张可改为 9 |
| 告警间隔 | **10 秒** | 同类告警最小间隔 |
| 排序器 | 5 / 0.4~0.5 | 按 WEB 字段填写 |

5. 可选：检测区域（ROI）、布防时段
6. **启用** 任务

建议逐步扩容：**4 → 8 → 16 → 32 路**，每档观察 30 分钟 GPU 利用率。

---

## 6. 查看告警

1. **告警事件** 列表按时间筛选
2. 点开事件查看：**抓拍图**、关联摄像头、置信度
3. 若配置了录像，可回放告警片段

通知渠道（钉钉、Webhook 等）在系统通知配置中设置（PoC 可选）。

## 7. 日常运维

### 7.1 服务状态

```bash
cd ~/easyaiot-deploy && ./scripts/deploy/verify.sh
docker ps
watch -n 1 nvidia-smi
```

### 7.2 重启模块

```bash
cd ~/easyaiot/.scripts/docker
# 按上游文档重启指定 compose 项目
sudo ./install_linux.sh restart   # 若上游脚本支持
```

### 7.3 日志

```bash
ls ~/easyaiot/.scripts/docker/logs/
docker logs -f <容器名>
```

## 8. 3060 PoC 容量参考

| GPU | 参考路数 | 备注 |
|-----|----------|------|
| RTX 3060 12GB | ~32 路 | 抽帧间隔 3、720p 算法流 |
| RTX 4090 | 更高 | 可用于压测上限 |

200 路生产估算：`ceil(200 / 单卡实测路数)` 张 GPU。
