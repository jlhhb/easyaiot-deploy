## 2. 架构与端口

| 模块 | 技术 | 默认端口 | 说明 |
|------|------|----------|------|
| WEB | Vue 3 | **8888** | 管理界面（主入口） |
| DEVICE | Java | **48080** | 设备、告警、网关 API |
| AI | Python | **5000** | 模型推理 |
| VIDEO | Python | **6000** | 视频拉流、算法任务 |
| Nacos | — | 8848 | 注册/配置中心 |
| MinIO | — | 9000/9001 | 对象存储 |
| Redis / PG / Kafka 等 | — | 见上游文档 | 中间件 |

防火墙示例（内网 PoC）：

```bash
sudo ufw allow 8888/tcp
sudo ufw allow 48080/tcp
sudo ufw allow 5000/tcp
sudo ufw allow 6000/tcp
```

## 3. 部署前准备

### 3.1 安装基础依赖

```bash
sudo apt update
sudo apt install -y git curl ca-certificates
```

### 3.2 安装 Docker（≥ 29）与 Compose（≥ 2.35）

按 [Docker 官方文档](https://docs.docker.com/engine/install/ubuntu/) 安装。验证：

```bash
docker --version
docker compose version
```

### 3.3 安装 NVIDIA 驱动与 Container Toolkit

```bash
nvidia-smi   # 确认驱动正常

# Container Toolkit（以 NVIDIA 官方文档为准）
sudo apt install -y nvidia-container-toolkit
sudo nvidia-ctk runtime configure --runtime=docker
sudo systemctl restart docker

docker run --rm --gpus all nvidia/cuda:12.0.0-base-ubuntu22.04 nvidia-smi
```

### 3.4 释放 GPU（若被其他服务占用）

安装 AI 模块前，确保至少一张 GPU 有足够显存。若存在 vLLM 等占满显存的进程，需先停止：

```bash
nvidia-smi          # 查看占用进程
# 示例：sudo systemctl stop vllm
```

## 4. 一键安装（推荐）

```bash
git clone git@github.com:jlhhb/easyaiot-deploy.git
cd easyaiot-deploy

# 1. 环境预检（root，配置镜像/Buildx/NVIDIA）
sudo ./scripts/deploy/preflight.sh

# 2. 全量安装（克隆上游 EasyAIoT 并执行官方 install）
./scripts/deploy/install.sh

# 3. 验证
./scripts/deploy/verify.sh
# 或指定 IP：EASYAIOT_HOST=192.168.1.10 ./scripts/deploy/verify.sh
```

### 4.1 环境变量

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `EASYAIOT_DIR` | `~/easyaiot` | 上游源码目录 |
| `EASYAIOT_REPO` | GitHub 官方仓库 | 可改为 Gitee 镜像 |
| `EASYAIOT_BRANCH` | `main` | 分支 |

Gitee 示例：

```bash
export EASYAIOT_REPO=https://gitee.com/soaring-xiongkulu/easyaiot.git
./scripts/deploy/install.sh
```

## 5. 本仓库脚本说明

### 5.1 `preflight.sh`

以 root 执行，完成：

- 安装 **Docker Buildx**（AI/VIDEO/WEB 构建必需）
- 写入 `/etc/docker/daemon.json`：
  - 镜像加速：`docker.m.daocloud.io`（主）、`docker.1ms.run`、`hub.rat.dev`
  - NVIDIA 默认 runtime
- 写入 `/etc/apt/.easyaiot_mirror_configured=skip`（跳过中间件安装时的 apt 镜像交互）
- 当前用户加入 `docker` 组

### 5.2 `install.sh`

- 克隆/更新上游 EasyAIoT
- Patch 安装脚本中的 Docker 镜像地址
- 调用 `easyaiot/.scripts/docker/install_linux.sh install`

### 5.3 `verify.sh`

- `nvidia-smi`、容器列表
- HTTP 探测 WEB/DEVICE/AI/VIDEO/Nacos
- 调用上游 `install_linux.sh verify`

## 6. 手动安装（备用）

若脚本失败，可手动执行：

```bash
git clone https://github.com/soaring-xiongkulu/easyaiot.git ~/easyaiot
cd ~/easyaiot/.scripts/docker
sudo ../path/to/easyaiot-deploy/scripts/deploy/preflight.sh
sudo ./install_linux.sh install
./install_linux.sh verify
```

## 7. 常见问题

| 现象 | 原因 / 处理 |
|------|-------------|
| 拉镜像超时 | 执行 `preflight.sh` 配置 DaoCloud；避免失效的内网 HTTP 代理 |
| `buildkit` / `buildx` 报错 | 安装 Buildx：`sudo ./scripts/deploy/preflight.sh` |
| 中间件安装卡在 apt 镜像选择 | 确认 `/etc/apt/.easyaiot_mirror_configured` 内容为 `skip` |
| AI 看不到 GPU | 检查 NVIDIA 驱动、Container Toolkit、`daemon.json` 中 nvidia runtime |
| WEB 构建卡在 pnpm | 等待或 `docker compose` 日志排查；必要时重试 install |
| 模型管理显示「未部署」但服务在跑 | 部署后调用 API 更新模型状态（见用户手册） |
| 端口冲突 | `ss -tlnp \| grep 8888` 查占用 |

日志目录：

```bash
ls -lt ~/easyaiot/.scripts/docker/logs/
```

## 9. 已部署环境安全操作

| 允许 | 禁止 |
|------|------|
| `verify.sh`、`poc-functional-test.sh` | 重复 `install.sh`（会重建容器） |
| WEB/API 创建摄像头与任务 | 无备份修改 `daemon.json` |
| `task/{id}/stop` 停止 PoC 任务 | 批量 `docker compose down` |

部分容器 Docker 健康检查显示 `unhealthy` 但 HTTP 200，属探针配置问题，**勿盲目重启**。
